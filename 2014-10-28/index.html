Vanilla JS Web Component


<template>
    <p>Hello World!</p>
</template>
<script>(function() {
    var thisDoc = document.currentScript.ownerDocument;

    var element = Object.create(HTMLElement.prototype);
    element.createdCallback = function() {
        var template = thisDoc.querySelector('template').content;
        // TEMPLATE
        var clone = document.importNode(template, true);
        // SHADOW DOM
        this.createShadowRoot().appendChild(clone);
    };
    // CUSTOM ELEMENT
    document.registerElement('hello-world', { prototype: element });
})();</script>
  • Déclaration en pur JavaScript
  • Exemple simple Hello World
  • Déclaration complexe

Language

Dart est un langage structuré, flexible, qui permet d'améliorer la maintenabilité des applications webs. Le langage est aussi open source, et depuis peu, c'est un standard ECMA.

Platform

  • Client and server
  • Virtuals Machines
  • Dependency manager
  • Javascript transpiler
  • IDE plugins
Dart est aussi une plateforme de développement complète. On ne verra que la partie client, mais à savoir que Dart peut aussi fonctionner en mode serveur. Dart tout comme JS est executé dans une VM, vous remarquez le 's' car les VM client est serveur sont différentes en terme d'api disponible. De nombreuse apis sont disponibles via pub, le gestionnaire de dépendences. Pour des raisons de comptabilité, le code est compilable en JS pour des performances équivalente. Et enfin, les IDES possèdent des plugins de développement.

Lorsque l'on souhaite afficher ou non, un morceau de html en fonction d'une condition, on va se servir de template if. Dans cet exemple, on affiche un texte différent en fonction que le nombre soit pair ou impair.

{{number | toHex | toUpperCase}}
            

@CustomTag('hex-number')
class HexNumber extends PolymerElement {

Un filtre est une fonction de transformation qui va prendre un objet A en entrée et le transformer en un objet B. Par exemple, la transformation d'un nombre en sa représentation hexadécimale.

Extending other elements


<polymer-element name="click-counter" extends="button" on-click="{{increment}}">
    <template>Click me: {{count}}</template>
    <script type="application/dart" src="click_counter.dart"></script>
</polymer-element>
            

@CustomTag('click-counter')
class ClickCounter extends ButtonElement with Polymer, Observable {
  ClickCounter.created() : super.created() {
    polymerCreated();
  }

  @observable int count = 0;

  increment() => count++;
}
            

<button is="click-counter"></button>
Il est aussi possible d'étendre des composants html classique comme un bouton. Dans sa déclaration, on note un extends. On retrouve le ckick handler. Comme Dart est un langage objet, extends ButtonElement. Le with permet d'indiquer que l'on va utiliser des mixins Polymer et Observable. Sans rentrer dans les détails, un mixin est une alternative à l'héritage multiple qui permet d'utiliser des comportemtns de l'objet. Coté constructeur petite contrainte, on va devoir apppeler, la méthode 'polymerCreated' qui fait partie du cycle de vie d'un composant polymer. A part ça, rien de nouveau. Et enfin, dans son utilisation, là on a une différence importante : on l'utilise comme un bouton classique à la différence de l'attribut is.
